1. Busca no Supabase os dados do guide da Carolina no Rio
2. Usa Gemini para organizar esses dados em um roteiro estruturado
3. Salva o resultado na tabela `public.friend_guide_itinerary_items`

IMPORTANTE:
- Não inventar lugares
- Não inventar restaurantes
- Não inventar hotéis
- Não inventar experiências
- Usar exclusivamente dados reais do Supabase
- Não alterar schema agora
- Não criar nova tabela

- public.friends

- slug: rio-carol-dieckmann

LÓGICA DE NEGÓCIO:
1. Buscar o `friend_guide` pelo slug `rio-carol-dieckmann`
2. Buscar todos os `friend_guide_places` desse guide
3. Separar os itens por categoria:
   - hotel
   - restaurante / bar
   - outros tipos
4. Priorizar:
   - `is_highlight = true`
   - `ordem`
   - `curation_dimension`
   - `bairro`
5. Enviar para Gemini um payload estruturado
6. Pedir para Gemini devolver JSON puro, sem texto extra
7. O JSON deve conter:
   - 3 dias
   - area_focus por dia
   - morning
   - lunch
   - afternoon
   - dinner
   - night
   - cada item referenciando `friend_guide_places.id`
8. Salvar esse JSON estruturado em `public.friend_guide_itinerary_items`

REGRAS DO ROTEIRO:
- não repetir o mesmo lugar em dias diferentes
- usar os hotéis como sugestão geral, não necessariamente repetidos em todos os dias
