We need to improve the app-wide HERO / BACKGROUND IMAGE SYSTEM.

This is now a product-priority visual fix because the current static sunset image is making the whole app feel flat, repetitive, and overly tinted in one single color mood.

PROBLEM

Right now, the same warm Rio sunset background stays static for too long and appears across too many screens.
This creates three problems:

1. the app feels visually repetitive
2. screens lose their own identity
3. the whole product feels covered by the same orange/brown atmosphere

I do NOT want that.

OBJECTIVE

Create a rotating, premium, smooth background hero system that makes the app feel alive, calmer, more editorial, and more sophisticated.

CRITICAL RULES

1. Do NOT touch engine logic
2. Do NOT touch itinerary logic
3. Do NOT touch map logic
4. Do NOT redesign unrelated screens
5. Focus ONLY on background / hero image behavior
6. Prefer existing Supabase image sources if already available
7. If a screen has no good Supabase source yet, use a safe local fallback pool
8. Keep the implementation lightweight and stable
9. Do NOT make the motion distracting or cheap
10. Respect readability and contrast at all times

UX REQUIREMENTS

I do NOT want the image to stay static.
I want it to rotate smoothly.

But I also do NOT want it to change too fast and feel nervous.

Recommended behavior:
- background image rotates every 10 to 12 seconds
- crossfade transition around 1.2 to 1.8 seconds
- no aggressive zoom
- no excessive parallax
- no slideshow feeling
- calm, premium, editorial motion

VISUAL GOALS

The app should no longer feel like it is all covered by the same Rio sunset.

Different screens should feel related, but not identical.

We need better visual identity separation between:
- Home
- Destinos
- Roteiro
- Perfil
- other key app areas

The "Viagem" area already feels better differentiated.
Use that principle.

DESIGN PRINCIPLES

1. Each screen should use a background image pool, not a single fixed image
2. Each screen or context should have its own family of images
3. Images must stay subtle behind content
4. Overlay / gradient / blur must preserve readability
5. The app should feel dynamic but calm
