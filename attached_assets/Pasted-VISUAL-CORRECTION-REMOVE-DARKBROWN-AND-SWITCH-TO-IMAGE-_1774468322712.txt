VISUAL CORRECTION — REMOVE DARKBROWN AND SWITCH TO IMAGE-DRIVEN UI

IMPORTANT:
Do not redesign the entire app.
Do not change layout structure.
Do not change navigation.
This is a visual system correction only.

CURRENT PROBLEM:
The app is using a “darkbrown” tone as a base color across screens.
This is incorrect and makes the UI feel heavy, dated, and disconnected from the destination imagery.

GOAL:
The visual style must be driven by images, not by a fixed brown color.

==================================================
1. REMOVE DARKBROWN AS BASE COLOR
==================================================

- Remove any hardcoded dark brown backgrounds
- Do not use brown as the default UI base
- Do not apply brown overlays on screens

==================================================
2. IMAGE-DRIVEN BACKGROUND SYSTEM
==================================================

==================================================
