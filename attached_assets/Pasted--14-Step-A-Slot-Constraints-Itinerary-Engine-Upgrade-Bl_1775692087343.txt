This step adds real capacity limits to tarde and noite (which currently have none), and enforces a sunset-last ordering rule within tarde. Both are pure Step-5 changes. Steps 1-4, 6, and 7 are untouched.

Problem Being Fixed

Problem A1 — No tarde/noite caps

MANHA_CAP exists and is enforced. tarde and noite have no cap.

Current behaviour at lines 938-943:

rests.forEach((r, i) => {
  const p: PeriodoDia = i === 0 ? "almoco" : "noite";
  periodMap.get(p)!.push(r); // all rests[1..n] pile uncapped into noite
});

Current behaviour for tarde at lines 932-935:

if (finalOverflow.length > 0) {
  periodMap.get("tarde")!.unshift(...finalOverflow); // unlimited overflow from manha
}

A user who saves 5 restaurants in Ipanema gets: almoco: 1, noite: 4 — an itinerary with four dinner slots on one day.

A user whose manha overflows heavily gets a tarde with 6+ items — an afternoon with no realistic time to complete them.

Problem A2 — Sunset items are not last within tarde

classifyPeriodo() correctly assigns sunset-tagged items to tarde. But Step 5d (sortByProximity) sorts tarde by travel distance only. A sunset viewpoint at Arpoador (the premium Ipanema sunset experience) can end up first in tarde — at 14:00 — instead of last at 17:30-18:00 where it belongs. Sunset is a time-anchored event: it must always be the final item in the afternoon.

Current sortByProximity at lines 622-638 — no time-of-day awareness, only distance:

function sortByProximity(items: EnrichedPlace[]): EnrichedPlace[] {
  if (items.length <= 2) return items;
  const result: EnrichedPlace[] = [items[0]];
  const remaining = items.slice(1);
  while (remaining.length > 0) {
    const last = result[result.length - 1];
    let bestIdx = 0, bestDist = Infinity;
    remaining.forEach((item, i) => {
      const dist = travelMinutes(last.zone, item.zone)
                 + subZoneCompatibilityPenalty(getSubZone(last.zone), getSubZone(item.zone));
      if (dist < bestDist) { bestDist = dist; bestIdx = i; }
    });
    result.push(remaining[bestIdx]);
    remaining.splice(bestIdx, 1);
  }
  return result;
}

Change 1 — Add TARDE_CAP and NOITE_CAP constants

Location: Line 886, immediately after the existing MANHA_CAP.

BEFORE (line 886):

AFTER:

const MANHA_CAP:  Record<string, number> = { tranquilo: 2, moderado: 2, intenso: 3 };
const TARDE_CAP:  Record<string, number> = { tranquilo: 2, moderado: 3, intenso: 4 };
const NOITE_CAP:  Record<string, number> = { tranquilo: 1, moderado: 2, intenso: 2 };

Rationale for cap values:

tarde tranquilo: 2 — a relaxed day has at most 2 afternoon items (e.g. a museum + a viewpoint)
tarde moderado: 3 — standard day: 3 afternoon activities is realistic
tarde intenso: 4 — packed day, but 4 is the physical ceiling for a 3h afternoon window
noite tranquilo: 1 — one dinner only (tranquilo travelers don't do late night)
noite moderado: 2 — dinner + one bar/nightlife item (the classic Rio evening)
noite intenso: 2 — same ceiling; 3 evening venues is unrealistic even for intenso
