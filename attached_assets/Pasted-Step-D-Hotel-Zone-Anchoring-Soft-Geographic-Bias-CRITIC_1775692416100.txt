### 3.3 — Soft hotel bias during grouping

Inside grouping logic (where items are assigned to buckets):

Add scoring bias:

const hotelBias = (item: EnrichedPlace) => {
  if (!hotelZone) return 0;
  if (item.zone === hotelZone) return 3;
  if (Math.abs(item.zone - hotelZone) === 1) return 1;
  return 0;
};

When sorting items for assignment:

.sort((a, b) => {
  const scoreA = a.prefScore + hotelBias(a);
  const scoreB = b.prefScore + hotelBias(b);
  return scoreB - scoreA;
});

IMPORTANT:
- This is a SOFT bias only
- prefScore still dominates
- no filtering

---

### 3.4 — Ensure Day 1 uses hotel-friendly cluster

After grouping is complete:

Sort day clusters:

days.sort((dayA, dayB) => {
  const scoreDay = (day) => {
    if (!hotelZone) return 0;
    const matches = day.filter(p => p.zone === hotelZone).length;
    const adj = day.filter(p => Math.abs(p.zone - hotelZone) === 1).length;
    return matches * 2 + adj;
  };
  return scoreDay(dayB) - scoreDay(dayA);
});

Result:
→ Day with most hotel-zone items becomes Day 1
