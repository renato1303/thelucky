Neighborhood image must be tied to the neighborhood itself (not to restaurants or hotels).

1. If a neighborhood image exists in Supabase (recommended: a bairros or locations table)

3. Only use a generic fallback if both fail

Do NOT reuse restaurant/hotel images as neighborhood hero images.

Same neighborhood = same image everywhere

No randomness.
No variation per screen.

   getNeighborhoodImage(neighborhoodName)

   - fetch from Supabase if available
   - fallback to Google Places (deterministic query)
   - cache result (avoid flickering or variation)

4. Replace all ad-hoc image usage with this resolver
